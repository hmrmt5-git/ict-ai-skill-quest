# ICT・AI SKILL QUEST 要件定義

この文書は、ICT・AI SKILL QUEST レベル1試作版の確定要件を、学校の非エンジニア担当者にも確認しやすい形で整理したものです。優先度は「必須」「推奨」「将来拡張」で示します。

| No. | 要件 | 優先度 | 内容 |
|---:|---|---|---|
| 1 | プロジェクトの目的 | 必須 | 教職員と高校生が、ゲーム感覚でICT・AIの知識だけでなく判断力を高められるWebアプリを提供します。 |
| 2 | 利用対象者 | 必須 | 学校法人早稲田学園の教職員、わせがく高等学校・わせがく夢育高等学校・わせがくPURE高等学校の生徒を対象とします。 |
| 3 | `wasegaku.ac.jp`限定認証 | 必須 | Google Workspaceアカウントのメールアドレスが `@wasegaku.ac.jp` で、メール確認済みであることをサーバー側でも確認します。 |
| 4 | 初回ユーザー登録項目 | 必須 | メールアドレス、表示用ユーザー名、利用者区分、所属校、登録日時、最終更新日時を登録します。 |
| 5 | 表示用ユーザー名の重複禁止 | 必須 | 前後空白削除、Unicode NFKC正規化、大文字小文字差を吸収したうえで、全利用者間の重複を禁止します。 |
| 6 | ユーザー名の30日に1回の変更制限 | 必須 | 表示用ユーザー名は変更可能ですが、変更は30日に1回までとし、変更履歴を保存します。 |
| 7 | レベル1から5の構成 | 必須 | レベル1〜5をステージとして表示します。試作版ではレベル1のみ挑戦可能、レベル2〜5は「準備中」です。 |
| 8 | 下位レベル合格による次レベル開放 | 将来拡張 | 完成版では下位レベル合格後に次レベルを開放します。試作版では上位レベルは開放しません。 |
| 9 | レベル1試作版の範囲 | 必須 | 20問の試作用仮問題から10問を出題し、採点、解説、結果、合格演出、修了証の試作導線を提供します。 |
| 10 | ICT3問、AI6問、総合1問の出題構成 | 必須 | 1回の挑戦ではICT3問、AI6問、ICT・AI総合1問の合計10問を出題します。 |
| 11 | 5択、10・8・5・2・0点の採点 | 必須 | 各問題は5択単一選択式で、選択肢に10点、8点、5点、2点、0点を1つずつ割り当てます。 |
| 12 | 84点以上の合格基準 | 必須 | 100点満点中84点以上を合格とします。 |
| 13 | 未出題問題優先 | 必須 | 利用者ごとに過去に出題されていない問題を優先して選び、不足時は出題回数の少ない問題を優先します。 |
| 14 | 1日5回制限 | 必須 | 同一レベルへの挑戦は1日5回までです。開始時点で1回として計上し、途中終了しても戻しません。 |
| 15 | Asia/Tokyo基準 | 必須 | 挑戦回数の日付リセットは日本時間（Asia/Tokyo）の午前0時を基準にします。 |
| 16 | レベル1の15分制限 | 必須 | レベル1の制限時間は15分です。本番ではサーバー開始時刻を基準に残り時間を管理します。 |
| 17 | 回答直後の解説 | 必須 | 回答確定後、獲得点、最も適切な回答、理由、不足していた視点、学校生活での例を表示します。 |
| 18 | 回答得点別サウンド | 推奨 | サウンドON時は得点に応じた控えめな音を再生します。サウンドだけに結果理解を依存させません。 |
| 19 | サウンド初期値OFF | 必須 | 学校利用を考慮し、初期状態はサウンドOFFにします。 |
| 20 | ライトモード・ダークモード | 必須 | 利用者がライト/ダークを切り替えられ、設定を保存します。CSS変数等で一元管理します。 |
| 21 | 合格時のくす玉とファンファーレ | 推奨 | 合格時はくす玉、紙吹雪、垂れ幕、ファンファーレを軽量に表示します。動きを減らす設定に対応します。 |
| 22 | 合格後の再挑戦禁止 | 必須 | 合格済みレベルには再挑戦できません。サーバー側の合格記録で確認します。 |
| 23 | PDF修了証 | 必須 | 合格時にPDF修了証を発行します。試作版では印刷表示のプレースホルダーを用意し、本番はサーバー側で生成します。 |
| 24 | 修了証番号 | 必須 | `L1-2026-A-00001` 形式を基本に、サーバー側トランザクションで重複なく発行します。 |
| 25 | QRコード検証 | 必須 | 推測困難な証明書IDまたはトークンを含む検証URLを用意し、未認証者には最小限の情報だけ表示します。 |
| 26 | リアルタイム合格者数 | 必須 | 各レベルの合格者数を公開用集計データとして表示します。ブラウザ側で全件集計しません。 |
| 27 | レベル1合格者100人の達成メーター | 必須 | レベル1合格者100人を目標に、現在数、達成率、残り人数、進捗ゲージを表示します。 |
| 28 | 管理者機能 | 必須 | 利用者、問題、挑戦履歴、合格、修了証、集計、管理者権限を確認・管理できる画面を用意します。 |
| 29 | Google Sitesへの埋め込み | 必須 | GitHub Pages URLをGoogle Sitesへ埋め込めるようにし、ログインできない場合の別タブ導線を常時表示します。 |
| 30 | GitHub Pagesでの公開 | 必須 | フロントエンドはGitHub Pagesで公開し、リポジトリ名を含むサブパスでも動作する構成にします。 |
| 31 | アクセシビリティ | 必須 | キーボード操作、フォーカス表示、適切な見出し、色だけに依存しない表示、十分なコントラスト、動きを減らす設定に対応します。 |
| 32 | セキュリティ | 必須 | 秘密情報を公開せず、採点・合否・回数制限・修了証番号・管理者権限をサーバー側で検証します。 |
| 33 | 試作版で実装しない項目 | 必須 | レベル2〜5本番問題、全250問、レベル5特別バッジ完成版、学校別デザイン、個人ランキング、外部生成AI API連携は試作版対象外です。 |

## 現在の試作実装との関係

現在のリポジトリには、依存を抑えた静的TypeScriptのフロントエンド試作と、Firebase本番構成を前提とした設計資料を置いています。認証、採点、挑戦回数、修了証PDF/QR生成、管理者権限は、本番ではFirebase Authentication、Firestore、Cloud Functions、Cloud Storage等でサーバー側に実装します。
